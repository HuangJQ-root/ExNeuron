标签元数据结构体，用于描述标签的名称及其关联的数据值。

此结构体包含了标签的基本信息（如名称）以及其对应的数值（包括类型、值和精度）， 适用于需要处理带有详细信息的标签数据的情况。

Special usage of parsing tag address, e.g. setting length of string type, setting of endian.

Parameters
[in]datatagcontains all the information about the tag.
[in]optioncontain actions that the label can perform.
Returns
0 on success, -1 on failure.

Special usage of parsing tag address, e.g. setting length of string type, setting of endian.

该函数根据传入的数据标签的数据类型，从数据标签的地址中解析出相应的选项信息， 并将解析结果存储在 option 指向的结构体中。不同的数据类型有不同的解析规则， 主要包括字节长度、字符串类型、字节序等选项的解析。

Parameters
datatag指向 neu_datatag_t 类型的常量指针，代表要解析的数据源标签。
option指向 neu_datatag_addr_option_u 类型的指针，用于存储解析得到的选项信息。
Returns
函数的返回值表示解析操作的结果：
  • 0：表示解析成功。
  • -1：表示解析过程中出现错误，例如未找到预期的分隔符、解析出的选项值无效等。

测试标签是否具有指定的属性。

该函数用于检查给定的数据标签是否设置了特定的属性。它通过位运算检查标签的属性字段， 判断是否包含指定的属性类型。

Parameters
tag指向 neu_datatag_t 结构体的指针，表示需要测试的标签。
attribute需要测试的属性类型（枚举 neu_attribute_e 的一个值）。
Returns
如果标签具有指定的属性，则返回 true；否则返回 false。

◆ neu_tag_get_icd()

UT_icd* neu_tag_get_icd ( )

获取用于数据标签管理的UT_icd结构体指针。

此函数返回一个静态定义的UT_icd结构体的地址，该结构体包含了处理数据标签类型的 创建(create)、拷贝(copy)、释放(dtor)和清除(clear)操作的函数指针。 UT_icd（Universal Type Interface for Dynamic Arrays）是用于动态数组管理的一个接口， 它允许用户自定义这些操作以便于对特定类型的数据进行管理。

Returns
返回指向tag_icd的指针，即数据标签类型的UT_icd结构体。
