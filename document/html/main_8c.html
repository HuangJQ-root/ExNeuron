配置远程系统日志。

该函数用于配置远程系统日志功能，通过指定的主机名或IP地址和端口号建立UDP套接字连接。 如果提供的主机名为无效IP地址，则尝试将其解析为域名。

Parameters
host远程系统日志服务器的主机名或IP地址。
port远程系统日志服务器的端口号。
Returns
int 成功时返回0；如果创建套接字失败或无法解析主机名，则返回-1。
Note
在使用此函数之前，请确保已正确初始化了 g_remote_syslog_ctx 结构体。 此函数依赖于标准C库中的 socket, inet_addr, inet_pton, gethostbyname 等函数。

< 地址族（IPv4）

< 端口号，转换为网络字节序

< IP地址

◆ remote_syslog()

static int remote_syslog ( zlog_msg_t *  msg)
static

发送日志消息到远程系统日志服务器。

该函数接收一个日志消息结构体，并将其发送到预先配置好的远程系统日志服务器。 在发送之前，它会调整消息的优先级字段以匹配syslog协议的要求。

Parameters
msg指向包含日志消息信息的 zlog_msg_t 结构体的指针。
Returns
int 成功时返回0；理论上此函数不处理任何错误情况，总是返回0。
Note
  • 确保在调用此函数之前已经通过 config_remote_syslog 正确配置了 g_remote_syslog_ctx。
  • 此函数依赖于 sendto 函数来发送UDP数据报。
  • 日志消息的优先级字段由 syslog_priority 函数根据 msg->path 计算得出。
