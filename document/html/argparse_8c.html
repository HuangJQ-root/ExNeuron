加载配置文件并更新CLI参数结构体。

该函数尝试根据命令行参数找到配置文件的位置， 如果找不到则使用默认配置文件名。接着读取文件内容， 解析其中的JSON数据，并更新提供的neu_cli_args_t结构体。

Parameters
argc命令行参数的数量（包括程序名）。
argv命令行参数数组。
long_options长选项列表，用于解析配置文件路径。
opts短选项字符串，用于解析配置文件路径。
args指向neu_cli_args_t结构体的指针，用于存储解析后的参数。
Returns
返回0表示成功，非零值表示错误。
Note
-加载配置元素有： disable_auth ip port syslog_host syslog_port sub_filter_error

由strdup() 分配内存，必须在使用结束后释放。 strdup 返回的指针是动态分配的内存，因此需要调用 free() 释放它。

加载环境配置参数到命令行参数结构体

该函数从系统环境变量中读取配置参数，用于初始化 neu_cli_args_t 结构体及输出路径指针。 支持解析环境变量以设置守护进程模式、日志级别、配置目录等参数，具备错误处理和内存管理功能。

Parameters
[in,out]args指向 neu_cli_args_t 结构体的指针
  • [输入]：结构体初始值（如默认日志级别、默认目录路径）
  • [输出]：填充从环境变量解析后的配置参数
[out]log_level_out指向 char* 的指针，存储解析出的日志级别字符串
  • 若环境变量存在，通过 strdup 分配内存，需调用者释放
  • 若环境变量不存在，保持原有值（需确保调用前已正确初始化）
[out]config_dir_out指向 char* 的指针，存储解析出的配置目录路径
  • 内存管理规则与 log_level_out 一致
[out]plugin_dir_out指向 char* 的指针，存储解析出的插件目录路径
  • 内存管理规则与 log_level_out 一致
Returns
0 成功解析所有有效环境变量 -1 解析过程中出现错误（如环境变量格式非法、内存分配失败）
Note
1. 环境变量优先级高于默认配置，未设置的变量将保留结构体默认值
  1. 支持的环境变量列表：
    • NEU_ENV_DAEMON: 守护进程模式（"1"启用，"0"禁用）
    • NEU_ENV_LOG: 开发日志功能（"1"启用，"0"禁用）
    • NEU_ENV_LOG_LEVEL: 日志级别字符串（如"debug"|"info"）
    • NEU_ENV_CONFIG_DIR: 配置文件目录路径
  2. 如何设置环境变量
    • 通过 shell 设置环境变量: export NEURON_DAEMON=1
  3. 错误处理：
    • 非法环境变量值将输出错误信息到标准输出
    • 内存分配失败时（如 strdup 失败）将提前返回 -1

解析命令行参数以确定配置文件路径。

该函数通过getopt_long解析命令行参数，查找指定的配置文件选项（如-c或–config_file）， 并将找到的配置文件路径存储在传入的指针中。如果未找到相关选项，则不会修改传入的指针。

Parameters
argc命令行参数的数量（包括程序名）。
argv命令行参数数组。
long_options长选项列表，用于解析配置文件路径。
opts短选项字符串，用于解析配置文件路径。
config_file指向char*的指针，用于存储解析后的配置文件路径。
Note
-optind:指示下一个要处理的命令行参数在 argv 数组中的索引位置。 随着 getopt_long 对命令行参数的解析逐步增加其值，直到所有选项都被处理完毕。 在某些情况下，你可能需要多次解析相同的命令行参数数组，或者你想确保在下一次调 用 getopt_long 时从头开始解析。这时，将 optind 设为0可以达到这个目的

-此处非必要：除非计划在同一个执行流程中再次调用 getopt_long 来解析相同的参数列表。 更常见的做法是让 getopt_long 自动管理 optind 的值，而不需要手动干预。

< 检查是否为长选项'–config_file'

< 对于未知选项，忽略处理

< 重置getopt库的状态变量
